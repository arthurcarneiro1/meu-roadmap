### **Desafio 1 – Classe genérica simples**

**Problema:**
Crie uma classe genérica chamada `Caixa<T>` que armazene um objeto de qualquer tipo.

* Adicione um método `getObjeto()` para retornar o objeto.
* Crie um objeto `Caixa<String>` com o valor `"Olá Mundo"` e imprima-o.

**Pergunta lógica:** Como os Generics permitem criar classes que funcionam com qualquer tipo de dado sem perder a segurança de tipos?

---

### **Desafio 2 – Método genérico**

**Problema:**
Crie um **método genérico** chamado `imprimirArray` que receba um array de qualquer tipo (`T[]`) e imprima todos os elementos.

* Teste com um array de inteiros e um array de strings.

**Pergunta lógica:** Como métodos genéricos permitem operar com diferentes tipos sem duplicar código?

---

### **Desafio 3 – Generics com Collections**

**Problema:**

* Crie uma lista genérica `List<T>` chamada `itens`.
* Adicione elementos de qualquer tipo (por exemplo, `Integer` ou `String`) e escreva um método que retorne o **maior elemento** usando `Comparable<T>`.

**Pergunta lógica:** Como Generics combinados com `Comparable` permitem criar métodos universais que funcionam com diferentes tipos de dados ordenáveis?
