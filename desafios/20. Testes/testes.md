### **Desafio 1 – Teste unitário simples**

**Problema:**

* Crie uma classe `Calculadora` com método `somar(int a, int b)`.
* Escreva um **teste unitário com JUnit** que verifique se `somar(2, 3)` retorna `5`.

**Pergunta lógica:** Como escrever um teste unitário simples para verificar o comportamento de um método?

---

### **Desafio 2 – Teste de exceção**

**Problema:**

* Crie um método `dividir(int a, int b)` que lança `IllegalArgumentException` se `b == 0`.
* Escreva um teste unitário que verifique se a exceção é lançada corretamente quando `b` é zero.

**Pergunta lógica:** Como testar cenários que geram exceções?

---

### **Desafio 3 – Teste com Assertions avançadas**

**Problema:**

* Crie um método `List<Integer> filtrarPares(List<Integer> numeros)` que retorna apenas os números pares.
* Escreva um teste unitário que verifique:

  1. O tamanho da lista retornada
  2. Se todos os elementos são pares
  3. Se a lista original não foi alterada

**Pergunta lógica:** Como usar diferentes assertions (`assertEquals`, `assertTrue`, `assertNotNull`) para validar comportamentos complexos?
